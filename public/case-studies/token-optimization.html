O Problema

O CLAUDE.md é carregado em cada turno de conversa com o modelo. Todo token desnecessário é um custo que se multiplica por centenas de interações. O arquivo original misturava narrativa pessoal, documentação de usuário e justificativas — nenhuma dessas categorias instrui comportamento.

✕ Narrativa descritiva — o modelo não precisa saber a história do projeto. Contexto de perfil já vive nas memórias persistentes.
✕ Justificativas — explicar por que uma preferência existe não instrui o comportamento do modelo.
✕ Documentação de usuário — "Como Usar Este Espaço" é um README. Não é uma diretiva.
✕ Duplicatas com memória — email e contexto de perfil já existem no sistema de memórias, tornando essas linhas puro overhead.

Princípios Extraídos

P-01
CLAUDE.md ≠ README

O arquivo é lido pelo modelo em cada turno, não pelo humano. Cada frase deve instruir comportamento — não descrever contexto.

P-02
Memória armazena, CLAUDE.md ativa

Contexto estático (perfil, histórico, stack) vai para memórias persistentes. CLAUDE.md contém apenas diretivas de comportamento ativo.

P-03
Workflows explícitos superam implícitos

Mapear a sequência de skills e ferramentas garante consistência sem depender de instrução verbal repetida a cada sessão.

P-04
Tokens acumulam — sempre mensure

220 tokens × 500 conversas = 110.000 tokens desperdiçados. Otimização de contexto é uma disciplina de engenharia, não um detalhe estético.
